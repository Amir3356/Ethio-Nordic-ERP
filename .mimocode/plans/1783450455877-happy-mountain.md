# Session Management Feature Plan

## Requirement Summary

Administrators can view all active sessions system-wide in real time, including Fullname, email, device, and location metadata, and can forcibly terminate any session. Idle sessions automatically expire after a configurable timeout, and refresh tokens are rotated on each renewal.

## Current State Analysis

**Good news: 95% of this functionality already exists.** The backend (`SessionController`, `TokenStateService`, `CheckIdleSession` middleware, `TokenRefreshService`) and frontend (`Sessions/` components) are fully implemented with:

- Real-time session viewing (30-second polling)
- Full metadata: Fullname, email, device, browser, platform, IP, location
- Force-terminate individual sessions
- Force-terminate all sessions for a user
- Configurable idle timeout (5-480 min)
- Refresh token rotation on renewal

## Identified Issues

### 1. Table Structure Mismatch (Bug)
**File:** `client/AdminDashboard/User Management Sidebar/Sessions/SessionTable.tsx`

The table has **10 `<th>` headers** but only **9 `<td>` cells per row**. Headers at lines 53-54 are empty, and the "Time" and "Status" headers are missing.

Current headers (lines 47-55):
```

```

Data cells (lines 61-103):
```
Fullname+email | Device | Browser | Platform | IP | Location | Time ago | Status | Terminate
```

### 2. Email Not as Separate Column
The user specifically requested "Fullname, email, device, and location" as separate items. Currently, email is shown as a `<small>` sub-element inside the Full Name cell, not as its own column.
